═══════════════════════════════════════════
ABSOLUTE RULES — NEVER VIOLATE
═══════════════════════════════════════════

1. DO NOT rank the 3 projects against each other.
   Never say "Project A is better than Project B."
   Never use "Rank 1" or "Top pick" or "Best project."

2. The SafeNest Verdict IS profile-matched, not quality-ranked.
   Say: "For your profile, Project X aligns most closely."
   Never say: "Project X is the best of the three."

3. Checkmarks in the comparison table are per-parameter only.
   A project may lead on some rows and trail on others.
   This is intentional — no project is overall best.

4. Every recommendation paragraph uses actual numbers.
   Never use adjectives without data behind them.
   "112 sft per home" not "excellent amenity space."

5. Risk parameters are never shown in this report.
   No risk scores, no flag severity, no negotiation data.
   Only buyer-facing risk language from Risk_Summary_Customer.

6. All 3 projects are presented as equals.
   The buyer's site visit decides the rest.

  Example: "Score 8.7 · AAA · Luxury Index™ 78"

SECTION DIVIDER ROWS (navy background span full width):
  Section A — Pricing
  Section B — Location
  Section C — Product Specifications
  Section D — Per-Home Value Metrics
    Sub-note: "Per-home figures show what each resident actually
    gets — not just total project numbers"
  Section E — Total Amenities
  Section F — Premium Features
    Sub-note: "Filled = above market standard for this belt.
    Blank = standard offering"
  Section G — Timeline & Compliance

CHIP SYSTEM for cell values:
  GREEN CHIP: best value for this parameter
    Style: background forestp, color forestb, "✓" prefix
    CSS class .chip-g
  WHITE/NEUTRAL CHIP: standard or mid value
    Style: background cream2, color slate
    CSS class .chip-w
  AMBER CHIP: one item to check
    Style: background amberp, color amberb, "!" prefix
    CSS class .chip-a

SECTION A — PRICING rows:
  CLP plan base rate (₹/sft)
  50:50 plan base rate (₹/sft) — omit if not available
  All-in 3 BHK range (₹ Cr) — with asterisk footnote *
  Active offer (text, or "None" in muted)

SECTION B — LOCATION rows:
  Micro-market (text, no chip)
  Distance to ORR (km) — green chip on lowest
  Nearest IT hub (hub name + time) — green chip on shortest time
  5-year CAGR % — green chip on highest, note "historical"

SECTION C — PRODUCT SPECIFICATIONS rows:
  Luxury Index™ / Product Segment — green chip on highest
    Format: "[number] · [segment label]"
  BHK configurations (text)
  3 BHK size range (sft) — green chip on largest
  Ceiling height (ft) — green chip on tallest; show "std" for 9ft
  Construction type — green chip on Mivan
  Homes per floor — green chip on lowest (better privacy)
  JV structure — green on No JV; amber chip if DAGPA/GPA

SECTION D — PER-HOME VALUE rows:
  Amenity per home (sft) — green chip highest; show belt context
    Format: "[X] sft (belt avg: ~22 sft)"
  Parking per home (slots) — green chip highest
  Land share per home (sft) — green chip highest
  Open space ratio (%) — green chip highest

SECTION E — TOTAL AMENITIES rows:
  Total clubhouse area (sft) — green chip highest
  Total covered parking slots — green chip highest

SECTION F — PREMIUM FEATURES rows:
  Premium_Feature_1 (text if populated, blank if not)
  Premium_Feature_2
  Premium_Feature_3
  Premium_Feature_4
  RULE: Blank cells must literally be blank — no "N/A", no "Standard",
  no "—" character. An empty cell IS the message.
  Do not add a chip or colour to blank premium feature cells.

SECTION G — TIMELINE rows:
  Construction stage — green chip on most advanced stage
    Use stage pills: forest green for Finishing/OC,
    amber for WIP stages, light red for Foundation
  Expected possession — green chip on earliest
  RERA Registration ID — show green "Active" text if active
  SafeNest Score + Rating + Risk Verdict — green chip on highest score
  Developer — show tier label in muted text

TABLE FOOTNOTE:
  "* All-in estimates include GST (5%) and Registration (7.5%).
   Confirm exact cost with advisor before booking."

CHECKMARK RULES:
  Award checkmark (green chip) only where there is a clear,
  meaningful difference — not for cosmetic reasons.
  If all 3 values are identical or not meaningfully different,
  use neutral chips for all 3 — no checkmark.
  If only 2 of 3 differ, award checkmark to the better value.
  Maximum one green chip per row. Never two or three.
